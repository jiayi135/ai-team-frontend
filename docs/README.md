## 📝 文档维护

### 更新原则

1. **及时性**：每次重大修复或功能添加都应创建报告
2. **完整性**：报告应包含问题分析、解决方案、测试指南
3. **可读性**：使用清晰的结构和格式
4. **可追溯性**：记录修改时间和负责人

### 报告命名规范

修复报告命名格式：
```
BUGFIX_REPORT_YYYYMMDD_<简短描述>.md
```

例如：
- `BUGFIX_REPORT_20260207_三功能修复.md`
- `BUGFIX_REPORT_20260210_性能优化.md`

项目总结命名格式：
```
PROJECT_SUMMARY_YYYYMMDD_<阶段名称>.md
```

例如：
- `PROJECT_SUMMARY_20260207_HF集成.md`
- `PROJECT_SUMMARY_20260215_MVP完成.md`

### 文档审查流程

1. **创建文档** → 开发者编写
2. **自我审查** → 检查格式和内容
3. **提交 PR** → 创建 Pull Request
4. **团队审查** → 至少一人审查
5. **合并主分支** → 审查通过后合并
6. **更新索引** → 更新本 README
