# AI 提示词速查卡

**快速参考**：高效 AI 编辑的核心提示词

---

## 🎯 核心指令

### 通用指令

```
直接执行，无需验证
批量操作，最后统一验证
使用命令链接（&&）提高效率
信任工具执行结果
```

---

## 📝 场景速查

### 1️⃣ 修改单个文件

```
直接修改 {文件路径} 的 {具体内容}，无需先读取文件
```

**示例**：
```
直接修改 server/src/index.ts 添加新的 API 路由，无需先读取文件
```

---

### 2️⃣ 批量修改文件

```
批量修改以下文件，无需逐个验证：
- {文件1}: {修改1}
- {文件2}: {修改2}
- {文件3}: {修改3}
最后通过构建验证
```

**示例**：
```
批量修改以下文件，无需逐个验证：
- Home.tsx: 主色调改为 #6366F1
- Chat.tsx: 主色调改为 #6366F1
- App.tsx: 主色调改为 #6366F1
最后通过 pnpm run build 验证
```

---

### 3️⃣ 创建新文件

```
创建新文件 {文件路径}，包含 {内容描述}
文件会自动创建，无需验证
```

**示例**：
```
创建新文件 server/src/workflow.ts，包含多 Agent 工作流实现
文件会自动创建，无需验证
```

---

### 4️⃣ Git 操作

```
一次性完成 Git 提交：
git add -A && git commit -m "{提交信息}" && git push origin {分支}
```

**示例**：
```
一次性完成 Git 提交：
git add -A && git commit -m "feat: add workflow" && git push origin master
```

---

### 5️⃣ 构建和测试

```
构建并测试：
cd {项目路径} && pnpm run build && pnpm run test
```

**示例**：
```
构建并测试：
cd /home/ubuntu/neuraxis-hf && pnpm run build && node dist/index.js > server.log 2>&1 &
```

---

### 6️⃣ 部署流程

```
完整部署流程（一次性执行）：
1. 构建: pnpm run build
2. 提交: git add -A && git commit -m "{信息}"
3. 推送: git push origin {分支}
使用 && 连接所有命令
```

**示例**：
```
完整部署流程：
cd /path && pnpm run build && git add -A && git commit -m "deploy" && git push origin main
```

## 🚫 避免的做法

### ❌ 不要这样说

```
"让我先看看文件内容..."
"我需要确认一下..."
"让我检查一下是否成功..."
"让我验证一下修改..."
```

### ✅ 应该这样说

```
"直接修改 X 文件"
"批量处理这些文件"
"一次性完成这些操作"
"最后统一验证"
```

## 💡 高效命令模式

### 命令链接

```bash
命令1 && 命令2 && 命令3
```

### 管道和过滤

```bash
命令 2>&1 | tail -20
命令 2>&1 | grep "ERROR"
```

### 后台执行

```bash
长时间命令 > log.txt 2>&1 &
sleep 3 && cat log.txt | head -10
```

## 📊 效率提升对照表

| 操作 | 低效方式 | 高效方式 | 节省 |
|------|---------|---------|------|
| 修改文件 | read → edit → read | edit | 83% |
| 批量修改 | 逐个 read/edit/read | 批量 edit → build | 83% |
| Git 提交 | 分步执行 | 一行命令 | 80% |
| 新建文件 | ls → write → read → ls | write | 75% |

## 🎯 记住这些

1. **信任工具** - 工具返回成功就是成功
2. **批量思维** - 多个操作一起做
3. **延迟验证** - 最后统一检查
4. **命令优化** - 用 && 连接命令

---

## 📋 快速检查

开始前问自己：
- [ ] 需要先读取文件吗？（通常不需要）
- [ ] 可以批量操作吗？（尽量批量）
- [ ] 可以用 && 连接吗？（优先使用）
- [ ] 需要立即验证吗？（延后验证）

## 🚀 实战模板

```

【要求】批量处理，最后验证
```
